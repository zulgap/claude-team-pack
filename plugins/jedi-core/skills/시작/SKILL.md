# 시작 — 마스터 허브 현황 보기 (팀원용)

줄갭의 **모든 작업 현황이 모인 노션 마스터 허브**를 매번 링크 복사 없이 열어, 제품·정본 인덱스와 진행 현황을 한눈에 정리하고 "오늘 무엇부터" 를 제시한다.

> 팀원용 경량 버전: 개발(코드·git·배포) 연계는 제외하고 **"허브 현황 보기"** 에 집중한다.

## 마스터 허브 (SSOT)

- 페이지 ID: `376efa2f2c3b80199b21f8026431f422`
- 성격: 링크 허브 (모든 제품·정본 인덱스). **읽기 전용 — 내용 수정 금지.** (정본은 각 하위 페이지에 있다)

## 실행 순서

0. **🩹 끊긴 작업 이어받기 (해당할 때만 — 아니면 바로 1번으로)**

   대화가 길어지면 중간에 끊기는 일이 있다. 그때는 **`/저장`을 누를 기회가 없어 기록이 아예 없다.** 그래서 이어받기는 *끝낼 때*가 아니라 **시작할 때** 한다.

   **이럴 때 실행**:
   - "끊겼어 / 이어서 하자 / 아까 하던 거" 라고 말함
   - 저장 안 된 변경사항이 있는데 이번 대화에서 만든 게 아님
   - `~/.claude/specs/` 에 **오늘 날짜 `*autohandoff*.md`** 가 있음 (= 끊김의 직접 증거)

   **찾아볼 곳 — 위에서부터, 전부 실제로 열어본다 (기억·추측 금지)**:

   | # | 어디 | 무엇이 나오나 |
   |---|---|---|
   | 1 | `ls -t ~/.claude/specs/*handoff*.md \| head -3` | **정식 핸드오프** — 이전에 `/저장`으로 남긴 진입점 |
   | 2 | `ls -t ~/.claude/specs/*autohandoff*.md \| head -3` | **자동 스냅샷** — 끊기기 직전 발화 목록 |
   | 3 | 지난 대화 검색 (episodic-memory) | **왜 그렇게 정했는지** — 파일에 안 남는 것 |
   | 4 | `git status` / `git log --oneline --since=midnight` | 하다 만 것 / 끊기기 전 결과물 |

   **규칙**:
   - 1번과 2번은 **역할이 다르다** — 1=정상 종료 시 정리본, 2=끊김 시 자동본. **둘 다 있으면 1번이 우선**, 2번은 그 이후 구간을 메우는 데 쓴다
   - 되살린 내용에는 **출처를 붙인다** (`[복구: 자동 스냅샷]`). 확인된 사실과 섞지 말 것
   - 안 나온 건 **추측하지 말고** `⚠️ 미확인` 으로 남긴다
   - **먼저 "이런 상태였습니다" 요약 보고 후, 이어서 할지 확인받고 진행** — 바로 파일부터 고치지 말 것
   - 아무 데서도 안 나오면 **"복구 실패 — 다시 알려주세요"** 라고 솔직히 말한다 (지어내지 말 것)

1. **마스터 허브 fetch**
   - Notion MCP `notion-fetch` 호출: `id = 376efa2f2c3b80199b21f8026431f422`
   - 실패 시: 링크만 제시하고 보고 (노션 권한이 없으면 사장님께 해당 페이지 공유를 요청).

2. **현황 파악**
   - 제품/섹션 + 각 제품의 **진행 현황 스냅샷**(✅ 배포 완료 / ⬜ 구현 예정) 추출.

3. **요약 출력** — 아래 형식으로 제시:

   ```
   🧭 마스터 허브 로드 완료

   📦 제품 현황
   - {제품명}: {한 줄 현황}
   - ...

   📌 오늘 후보
   1. {제품} — {다음 작업}
   2. ...

   어느 것부터 진행할까요?
   ```

## 함정

- 마스터 허브는 **링크 허브** — 내용을 여기서 복제·수정하지 말 것 (정본은 각 하위 페이지에서만).
- 이 스킬은 **읽기 전용** — 스냅샷을 임의로 고치지 않는다.
- 노션 페이지가 안 열리면 = 권한 문제 → 사장님께 마스터 허브 공유 요청.
