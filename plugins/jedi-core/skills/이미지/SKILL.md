# 이미지 v2 — 생성·변형·시리즈 (팀원용)

원하는 그림을 말하면 줄갭 백엔드가 **GPT Image 2**(실패 시 Gemini 자동 폴백)로 만들어 준다. 키 입력·설치 불필요 — 네 제디 토큰으로 회사 작업공간에 자동 저장된다.

> 팀원용: 프롬프트는 **한국어로 그대로** 말해도 된다(백엔드가 알아서 영문 변환).
> **제디(jedi) 도구가 활성일 때만 동작** — 개인 `JEDI_TOKEN` 등록 필요(없으면 `/시작`의 도구 안내 참고).

## 입력 유형 판별 (먼저 이것부터)

사용자 요청을 아래 4가지 중 하나로 분류하고 해당 섹션의 순서를 따른다:

| 유형 | 신호 | 섹션 |
|---|---|---|
| **① 새로 생성** | "~그려줘", "포스터/배너/썸네일 만들어" (참조물 없음) | A |
| **② 문서 → 인포그래픽** | 회의록·보고서·글을 주며 "이걸 이미지/카드로" | B |
| **③ 내 사진 변형·합성** | 로컬 파일 경로/첨부 + "이 사진으로", "얼굴 유지", "합성" | C |
| **④ 씬 시리즈 (인물 고정)** | 대본·스토리 + "여러 장", "같은 인물로 장면들" | D |

공통 규칙:
- **비율**은 픽셀이 아니라 종류로: 정사각 `1:1`(프로필·SNS) / 세로 `9:16`·`3:4`(표지·포스터·스토리) / 가로 `16:9`·`4:3`(배너·썸네일·슬라이드). 모르겠으면 `1:1`.
- **생성 직전 1줄 미리보기**: `컨셉 / 비율 / 장수` 확인. (사용자가 이미 "만들어"라고 명확히 했으면 바로 진행)
- **회사(tenant)는 네 토큰으로 자동 결정** — `tenant_id` 등 회사 식별 인자는 **절대 넘기지 말 것**(서버가 강제).
- 유료 API 호출(장당 비용 발생) — **3장 이상은 반드시 장수·비용을 먼저 고지**하고 진행.

## A. 새로 생성 (텍스트 → 이미지)

1. 무엇을 그릴지(프롬프트) + 비율 파악.
2. **`ext_generate_image` 도구 호출**:
   - `prompt`: 그릴 내용 (한국어 그대로)
   - `aspect_ratio`: `1:1` / `9:16` / `3:4` / `16:9` / `4:3`
   - `n`: 장수 (기본 1, 최대 8)
3. 반환된 **`image_url`** 전달: `✅ 이미지 생성 완료 — {비율}, {장수}장\n{image_url}`

## B. 문서 → 인포그래픽/카드 (기획은 네가, 그림은 백엔드가)

회의록·보고서·긴 글을 이미지로 만들 때는 **바로 그리지 말고 기획 1단계**를 거친다:

1. 문서를 읽고 **핵심 3~5개를 구조화** (제목 1 + 포인트 3~5 + 결론/CTA).
2. 구조를 사용자에게 1줄 확인: "이 구성으로 인포그래픽 만들까요? — {제목} / {포인트들}".
3. 확인되면 구성을 프롬프트에 녹여 `ext_generate_image` 호출 (레이아웃 지시 포함: "상단 큰 제목, 중앙 3개 아이콘+짧은 라벨, 하단 결론 배너" 식).
4. ⚠️ **한글 본문 텍스트는 모델이 자주 깨뜨린다** — 이미지 안에는 **짧은 라벨만** 넣고, 긴 문장은 이미지 밖(캡션/본문)으로. 1차 결과는 *배경·구도·색감 시안*으로 보고 최종 문구는 디자인에서 재식자 권장.

## C. 내 사진 변형·합성 (로컬 파일 → 업로드 → 변형)

로컬 PC 사진도 지원한다. **2단계**: 먼저 URL로 만들고, 그 URL로 변형한다.

1. **업로드** (파일 바이트는 HTTP 직송 — 대화 컨텍스트에 base64를 절대 넣지 말 것):
   ```bash
   node <이 스킬 폴더>/scripts/upload-image.mjs "C:/경로/사진.jpg"
   ```
   - 마지막 줄 출력 = `image_url`. png/jpeg/gif/webp만, 6MB 이하.
   - 실패 메시지가 한국어로 안내됨 (토큰 없음/형식/크기).
2. **변형 호출** — `ext_generate_image`에 URL 전달:
   - `mode: "edit"` (그 사진을 직접 수정: 배경 교체, 옷 변경, 보정) + `input_images: [{ url: "<업로드 URL>", role: "edit_target" }]`
   - `mode: "style_ref"` (스타일만 차용해 새 그림) + `role: "reference"`
   - `mode: "compose"` (여러 장 합성 — 인물 A + 배경 B) + `input_images` 2장 이상, 각각 role 지정
   - 얼굴·정체성 유지가 중요하면 `preserve_fidelity: true` **필수**.
3. 결과 `image_url` 전달. 원본 대비 뭘 바꿨는지 1줄 요약.

## D. 대본 → 씬 시리즈 (같은 인물로 여러 장면)

스토리·대본을 여러 장의 장면 이미지로, **인물이 장면마다 바뀌지 않게**:

1. **씬 분해**: 대본을 읽고 씬 목록(장면·구도·감정) 3~8개로 나눠 사용자 확인. **장수 = 비용 고지 필수**.
2. **캐릭터 시트 먼저 1장**: 주인공 외형 묘사로 정면 캐릭터 시트를 `ext_generate_image`로 생성 (또는 사용자가 준 사진을 C-1로 업로드).
3. **각 씬 생성**: 캐릭터 시트의 `image_url`을 모든 씬 호출에 고정 레퍼런스로:
   - `mode: "style_ref"` + `input_images: [{ url: "<캐릭터시트 URL>", role: "reference" }]` + `preserve_fidelity: true`
   - `prompt`: 해당 씬 묘사 (인물 외형은 "레퍼런스와 동일 인물" 명시)
4. 씬 번호와 함께 URL 목록 전달. 특정 씬만 재생성 요청 가능.

## 안 되는 경우

- "제디 도구가 안 보임 / `ext_generate_image` 없음" → `JEDI_TOKEN` 미등록. install 재실행 또는 사장님께 토큰 요청.
- 업로드 415 오류 → png/jpeg/gif/webp만 가능 (파일 **내용** 기준 — 확장자만 바꾼 파일은 거부됨). HEIC(아이폰)는 jpg로 변환 후 재시도.
- 업로드 6MB 초과 → 해상도 줄여서 재시도.
- 한글 텍스트 깨짐 → B-4 참고 (짧은 라벨만, 재식자 권장).
