# 스킬 — 지금 쓸 수 있는 도구 한눈에 보기 (팀원용)

줄갭 팀원이 **지금 이 PC에서 실제로 쓸 수 있는 스킬(명령)**을 설명과 함께 목록으로 보여주고, 그 자리에서 하나 골라 바로 실행하게 한다. 새 스킬이 추가되면 이 목록에 **자동으로** 나타난다 (설치된 것만 보이므로 항상 최신 = 업데이트 자동 반영).

## 핵심 원리 (읽고 그대로 따를 것)

이 스킬은 **파일이나 플러그인 폴더를 스캔하지 않는다.** Claude Code가 세션 시작 시 이미 "사용 가능한 스킬 목록"을 컨텍스트에 넣어 준다. 그 목록 = **이 PC에 설치된 것 = 이 팀원이 보유한 것**이라, 그걸 그대로 쓰면 테넌트·역할별로 자동 정확하다. `/스킬`이 할 일은 그 목록에서 **줄갭 팀 스킬만 골라** 사람이 읽기 쉽게 정리해 보여주는 것뿐이다.

### 1단계 — 줄갭 팀 스킬 선별

사용 가능한 스킬 목록에서 아래 규칙으로 **줄갭 팀 스킬만** 추린다:

- ✅ **포함**: 이름에 `:`(콜론)이 없는 스킬 = 줄갭 팀팩/개인 스킬. (예: `시작`, `저장`, `이미지`, `키워드`, `인입`, `썸네일` 등 jedi-core 코어 스킬 + 회사 전용 스킬 + 개발자용 `start-dev`·`wrapup-dev`)
- ❌ **제외 A — 콜론 플러그인**: 이름에 `:`가 붙은 것 (`superpowers:*`, `anthropic-skills:*`, `figma:*`, `Notion:*`, `codex:*`, `ralph-loop:*` 등). 이건 Claude Code 기본 플러그인이라 팀원 업무와 무관.
- ❌ **제외 B — Claude Code 내장 시스템 명령**: 영어 소문자 한 단어 시스템 명령 (`init`, `review`, `security-review`, `verify`, `run`, `code-review`, `simplify`, `loop`, `schedule`, `update-config` 등). 줄갭 스킬이 아님.
- 🆕 **새 스킬 자동 포함**: 위 예시에 없는 스킬이라도 **콜론 없고 + 내장 시스템 명령이 아니면** 줄갭 팀 스킬로 보고 포함한다. (새로 배포된 스킬이 목록에 자동으로 뜨는 게 이 명령의 핵심 가치.)

> **선별 규칙의 안전장치 (드롭보다 표시)**: 이 규칙은 완벽한 분류가 아니라 **"빠뜨리지 않기"**가 목적이다. 판단이 애매한 스킬은 **절대 제외하지 말고 "🆕 기타" 칸에 넣어** 사용자가 보게 한다.
> - **콜론 없는 영어 이름 스킬**(예: 앞으로 외국인 팀원이 만든 팀 스킬)이 나오면 → known 내장 명령 목록(init·review·verify 등)에 없는 한 **팀 스킬로 보고 "🆕 기타"에 표시**. (드롭 금지.)
> - 내장 명령 열거는 **완결 목록이 아니다** — 여기 없는 새 내장 명령이 섞여 들어와도 최악의 경우 "🆕 기타"에 한 줄 더 뜰 뿐(경미한 노이즈), 팀 스킬을 잃지 않는다. 이 트레이드오프(과표시 허용 / 미표시 금지)가 의도된 설계다.

### 2단계 — 설명은 각 스킬 자체 description에서

각 스킬의 한 줄 설명은 **그 스킬 자체의 description을 요약**해서 쓴다. **이 파일에 설명을 베껴 하드코딩하지 말 것** — description이 SSOT라, 스킬이 업데이트되면 설명도 자동으로 최신이 된다. (team-guide.md 정적 목록이 드리프트 나는 것과 같은 함정을 피한다.)

### 3단계 — 카테고리로 묶어서 표시

성격이 비슷한 것끼리 묶는다. 아래는 **jedi-core 코어 스킬** 기준 예시 매핑이다. 그 외 설치된 스킬(회사 전용 스킬 포함)은 **각자의 description으로 성격을 판단**해 알맞은 칸 또는 "🏢 회사 전용" / "🆕 기타"에 배치한다. (여기 카테고리 표에 회사 전용 스킬 이름을 하드코딩하지 않는다.)

| 카테고리 | jedi-core 예시 |
|---|---|
| 🚀 작업 흐름 | `시작`, `저장` |
| ✍️ 콘텐츠·이미지 | `이미지`, `썸네일` |
| 🔎 자료·조사 | `키워드`, `인입` |
| 🏢 회사 전용 | (설치돼 있으면 여기 — description으로 판단) |
| 🛠 개발 (dev 팀원만) | `start-dev`, `wrapup-dev` |
| 🆕 기타 | 위에 안 맞는 새 스킬 |

### 4단계 — 사용자에게 보여줄 형식

```
🧰 지금 쓸 수 있는 줄갭 스킬

🚀 작업 흐름
- /시작 — (그 스킬 description 한 줄 요약)
- /저장 — ...

✍️ 콘텐츠·이미지
- /이미지 — ... (제디 토큰 필요)
- /썸네일 — ... (제디 토큰 필요)

🔎 자료·조사
- /키워드 — ...
- /인입 — ... (제디 토큰 필요)

번호나 이름을 말씀하시면 바로 실행합니다. (예: "이미지" 또는 "3번")
새 스킬이 추가되면 여기에 자동으로 나타납니다.
```

- 각 줄 형식: `- /이름 — 한 줄 설명 (조건: 제디 토큰 필요 / 담당 전용 등, 있을 때만)`
- 조건 괄호는 **있을 때만** 붙인다 (없으면 생략, 군더더기 방지).

### 5단계 — 선택 즉시 실행

사용자가 스킬 **이름이나 번호를 말하면 되묻지 말고 그 스킬을 Skill 도구로 바로 호출**한다.
- 인자 없이 시작하는 스킬(`시작`·`저장` 등) → 즉시 실행.
- 인자가 필요한 스킬(글 주제, 노션 링크, 이미지 프롬프트 등) → 그때 **필요한 입력 1가지만** 1줄로 묻고 실행. 미리 여러 개 몰아 묻지 말 것.

## 함정 / 규칙

- ❌ **설명을 이 파일에 하드코딩 금지** — 각 스킬 description이 SSOT (드리프트 방지).
- ❌ **파일시스템·플러그인 폴더 스캔 금지** — 컨텍스트의 "사용 가능 스킬 목록"이 이미 정확하다.
- ❌ **회사 전용 스킬 이름을 이 본문에 명명 금지** — 이 스킬은 전 테넌트 공유(jedi-core)라, 특정 회사 스킬명을 박으면 그 회사가 없는 다른 테넌트에도 노출된다. 회사 전용은 실행 시점의 사용 가능 목록에서 **동적으로** 잡아 "🏢 회사 전용"에 넣는다.
- ✅ **역할 자동 반영** — dev 팀원 PC엔 `start-dev`/`wrapup-dev`가 목록에 있고 staff엔 없다 (설치 팩이 다름). **목록이 곧 권한** — role 분기 코드 불필요.
- ✅ **목록이 비면 안내** — 줄갭 팀 스킬이 0개면 "설치된 줄갭 스킬이 없습니다. `/시작`으로 시작하거나 사장님께 문의하세요"로 안내.
