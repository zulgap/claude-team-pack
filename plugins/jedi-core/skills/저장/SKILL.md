# 세션 저장 (팀원용)

이번 작업을 ① **프롬프트 수집**(항상) ② **노션 팀 세션 저널 기록**(신호 있을 때) ③ **핸드오프 프롬퍼**(미완료일 때) 순으로 마무리한다.
목적: 무엇을·왜 했는지 구조화해 누적 → 검색·정리·인수인계가 쉽게 + 프롬프트를 데이터로 축적.

> 팀원용: 복잡한 정본 라우팅은 빼고 **"수집 → 5섹션 기록 → (미완료 시)핸드오프"** 에 집중한다.

## 0) 프롬프트 수집 (무조건 먼저 — 저널 여부와 무관)

이번 세션에서 친 프롬프트를 데이터 해자(`prompt_log`)로 결정론 수집한다. **적재 대상이든 아니든 항상 1회 실행.** 이 스킬 디렉토리의 `collect-prompts.js`를 Bash로:

```
node "<이 스킬 디렉토리>/collect-prompts.js"
```
- `<이 스킬 디렉토리>` = 스킬 호출 시 안내된 "Base directory for this skill" 경로.
- 출력 예: `프롬프트 N건 중 신규 X / 기존(멱등) Y / 실패 Z`. 실패·토큰없음이어도 **조용히 넘어가고** 저널은 정상 진행(수집은 부수).
- 멱등(turn_uuid) — 재실행해도 중복 안 쌓임. LLM 개입 0(결정론).

## 적재 여부 판단 (먼저 자문)

이번 작업에 **핵심 결정 / 만든 것(산출물) / 배운 점** 중 하나라도 있었는가?
- YES → 적재.
- NO (단순 질문·조회만) → "이번 작업은 기록 대상이 아닙니다" 1줄 보고 후 종료.

## 작성자 자동 확인 (적재 직전, 먼저)

공유 계정이라 "누가 했는지"는 개인 제디 토큰의 actor로 식별한다. 적재 직전 **딱 1회** Bash로 실행:

```
node "$HOME/.claude/plugins/marketplaces/zulgap-team-pack/resolve-staff.js"
```
- 위 경로가 없으면 `~/.claude/plugins` 아래에서 `resolve-staff.js`를 찾아 실행한다(플러그인 캐시 경로 환경차).
- 출력된 **이름 한 줄**을 아래 `작성자` 속성에 넣는다.
- 출력이 **비어 있으면**(토큰 없는 직원 / 미매핑) `작성자`는 **생략**한다. 스크립트 실패도 무시하고 저널은 정상 적재한다.

## 적재 방법

`notion-create-pages` 호출:
- **parent**: `{"type":"data_source_id","data_source_id":"67bf5daa-64e2-4745-85b6-c67e4a231b44"}`
  - (줄갭 "팀 세션 저널" DB — 마스터 허브 하위. 직원이 이 DB에 노션 권한이 있어야 적재됨)
- **properties**:
  - `세션` (title): 이번 작업 한 줄 제목
  - `작성자`: 위에서 얻은 이름 (비면 생략)
  - `date:날짜:start`: 오늘 날짜 (YYYY-MM-DD), `date:날짜:is_datetime`: 0
  - `유형`: 콘텐츠제작 / 조사 / 문서작성 / 기타 중 1개
  - `한줄 요약`: 이 작업에서 가장 중요한 것 1줄
  - `산출물`: 만든 것(링크·파일명)
- **content (body) = 표준 5섹션 고정**:
  ```
  ## 배경 / 요청
  무엇을 하려 했는가 (1~2줄)

  ## 핵심 결정 + 근거
  무엇을 정했고 왜

  ## 학습 · 함정
  하면서 알게 된 것, 다음에 조심할 것

  ## 산출물
  만든 콘텐츠 / 문서 / 링크

  ## 미해결 · 다음
  다음에 이어서 할 것
  ```

## 캡처 원칙 (노이즈 차단)

- **담아라**: 핵심 결정·근거 / 만든 것 / 배운 점.
- **빼라**: 도구 실행 로그 / 중간 과정 / 뻔한 것.
- **형식 일관성 = 매번 같은 5섹션** (나중에 검색·정리가 쉽다).

## 핸드오프 프롬퍼 (작업이 아직 안 끝났으면)

저널 기록 후 **"이 작업이 다 끝났나"** 를 자문한다. 아래 중 하나라도 해당하면 = 미완료 → 다음 세션이 바로 이어갈 **핸드오프 파일**을 만든다:
- ⬜ 만들던 게 중간에 끊김 (일부만 완성 / 검증 안 함)
- ⬜ 다음에 이어서 해야 할 게 남음
- ⬜ 누구 답·외부 자료를 기다리는 중

**전부 끝났으면** → 만들지 않고 "작업 완결 — 핸드오프 불필요" 1줄 보고.

미완료면 `~/.claude/specs/YYYY-MM-DD-handoff-<작업한단어>.md` 에 아래 5블록으로 작성:
```markdown
# 핸드오프 — <작업명>

## 한 줄
<무엇을 하면 이 작업이 끝나는가 — 1문장>

## 다음에 바로 할 것 (복붙 프롬프트)
<다음 세션 첫 발화로 그대로 붙여넣을 프롬프트>

## 지금까지 한 것
<완성된 것 / 결정한 것>

## 남은 것
<⬜ 남은 항목>

## 조심할 것
<손대면 안 되는 것 / 함정>
```
작성했으면 그 경로도 완료 보고에 포함한다.

## 완료 후

적재된 노션 페이지 URL(+미완료면 핸드오프 경로)을 보고한다. (텔레그램 알림은 보내지 않는다)
