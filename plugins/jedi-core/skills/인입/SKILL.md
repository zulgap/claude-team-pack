# 인입 — 자료를 디지털 트윈에 쌓기 (팀원용)

자료를 주면 줄갭 디지털 트윈이 알아서 **분류**하고, 어떤 객체(회사·종목·개념)에 연결할지 **매핑**하고, **3종 DB(기록·검색·관계)에 한 번에** 올린다.

> 팀원용: 복잡한 분류·매핑은 시스템이 한다. 너는 "무슨 자료인지"만 알려주면 된다.
> **제디(jedi) 도구가 활성일 때만 동작** — 개인 `JEDI_TOKEN` 등록 필요(없으면 `/시작`의 도구 안내 참고).

## 실행 순서

1. 사용자가 준 자료의 성격을 파악한다(강의·문서 / 작업·결정 / 종목·키워드). 모르겠으면 "자동"으로 둔다.
2. **필요할 때만** 한 번 확인(발화에 이미 드러나면 묻지 말 것):
   - "이 자료는 어떤 종류인가요?" → [강의·문서 / 작업·결정 / 종목·키워드 / 자동]
   - "검수 알림은 어떻게 받을까요?" → [즉시 / 매일 모아서 / 안 받음]
3. **`ingest_data` 도구 호출** (제디 MCP 브릿지 경유):
   - `category`: `knowledge`(강의·문서) / `judgment`(작업·결정) / `object`(종목·키워드) — 자동이면 생략
   - `text`: 원문 텍스트 / `object_name`: 객체명(종목·키워드일 때)
   - `notify`: `immediate`(즉시) / `daily`(매일 모아서, 기본) / `none`(안 받음)
   - **회사(tenant)는 네 토큰으로 자동 결정** — 인자로 지정하지 말 것.
4. 결과를 한 줄로 보고:
   - 정상: `✅ 인입 완료 — {분류}/{상태}` (+ 객체명이 있으면 함께)
   - 검수 필요: `🔎 매핑이 모호해 검수 큐로 보냈어요. 미니앱 검토탭에서 확정/거부해 주세요.`

## 첨부 파일을 인입할 때

PDF·워드 등 첨부는 먼저 본문 텍스트를 뽑은 뒤 그 텍스트를 `text`로 넘긴다(제디의 첨부 추출 도구 사용). 사진/스캔도 동일.

## 안 되는 경우

- "제디 도구가 안 보임" → `JEDI_TOKEN` 미등록. install 재실행 또는 사장님께 토큰 요청.
- **확정·삭제는 사람만** — AI는 초안까지. 모호한 매핑은 검수 큐로 보내고 미니앱 검토탭에서 확정한다.
- 다른 회사 자료를 임의로 지정하지 말 것 — 시스템이 네 토큰의 회사로 고정하며, 불일치 시 차단된다.
