# 키워드 — 황금키워드·황금질문 조사 (팀원용)

글을 쓰기 **전에** 부른다. "이 키워드로 글 쓰면 상위에 뜰까? 광고비는 붙을까?"(황금키워드) + "고객이 실제로 뭘 묻고 있나?"(황금질문)를 **네이버·지식iN 공식 API 숫자로** 답한다.

> **LLM 추정 0.** 모든 숫자는 네이버 API 원본값이고, 판정은 그 숫자에 대한 규칙일 뿐이다. 같은 키워드를 열 번 물으면 열 번 같은 답이 나온다.
> **제디(jedi) 도구가 활성일 때만 동작** — 개인 `JEDI_TOKEN` 필요(없으면 `/시작`의 도구 안내 참고).

## 절대 규칙

- **회사(tenant)는 네 토큰으로 자동 결정** — `tenant_id`는 **절대 넘기지 말 것**(서버가 강제).
- **숫자를 지어내지 말 것.** 도구가 `null`을 주면 "조회 실패"라고 그대로 말한다. 추측해서 채우면 그 순간 이 도구는 쓸모가 없어진다.
- **`entry`(진입)가 1차 축이다.** 단가가 아무리 높아도 노출이 안 되면 말짱꽝 — 사장님 방침.

## 사용 흐름

### 1) 사용자 발화에서 모드를 고른다

| 발화 | 모드 | seed |
|---|---|---|
| "검단 치과 키워드 조사해줘" / "이 키워드 써도 돼?" | `keyword` (기본) | 그 키워드 |
| "임플란트 마케팅 **주제로** 쓸 만한 키워드 찾아줘" / "○○ 업종 키워드 발굴" | `topic` | 그 주제 |
| "○○ 관련 **고객이 뭘 물어봐?**" / "황금질문 찾아줘" / "실제 질문으로 글 쓰고 싶어" | `question` | 그 주제·키워드 |

애매하면 **되묻지 말고 `keyword`로 먼저 돌린다.** 결과를 보고 사용자가 방향을 잡는 게 빠르다.
"질문/물어봐/궁금해하는 것" 류 발화만 `question`. 글감을 제대로 잡고 싶다는 발화면 **keyword + question 둘 다** 돌려도 좋다(각 1콜).

### 2) 도구 호출

```
mcp__jedi__ext_golden_keyword({ seed: "검단 치과", mode: "keyword", limit: 10 })
mcp__jedi__ext_golden_keyword({ seed: "임플란트", mode: "question", limit: 10 })
```

- `limit` 기본 10. **키워드당 발행량 조회 1콜**이라 20으로 올리면 그만큼 느리다. 사용자가 "많이 보여줘" 하지 않으면 기본값.
- 조회에 20~40초 걸릴 수 있다. 오래 걸린다고 중간에 다시 부르지 말 것.
- `question` 모드 반환: `gap_band`(미충족 갭) + `kin_total`(지식iN 질문 누적수=수요) + `blog_1m_count`(발행량=공급) + `items[]`(질문/출처/헤드키워드/상업마커).

### 3) 결과는 **5섹션 고정 보고서**로 보여준다

아래 5섹션을 **이 순서·이 헤더 그대로** 출력한다 (섹션 생략·순서 변경·신규 섹션 추가 금지 — 제디 스킬과 같은 포맷):

```
📊 키워드 조사 보고서 — 검단 치과 (2026-07-15)

① 한 줄 결론
조건부 — "치과"는 포화지만 "부평임플란트"가 진입 가능 + 단가 상위.

② 🥇 황금키워드 (진입 × 단가)
| # | 키워드 | 검색량/월 | CPC | 30일 발행 | 판정 |
|---|---|---|---|---|---|
| 1 | 부평임플란트 | 5,270 | 28,315원 | 33 | 🟢 황금 |
| 2 | 부천치과 | 7,850 | 36,760원 | 100+ | 🔴 포화 |
| 3 | 치과 | 241,600 | 19,480원 | 100+ | 🔴 포화 |

③ ❓ 황금질문 (수요 ÷ 공급 = 미충족 갭)
포화 (수요 487,888 ÷ 공급 100+)
1. 임플란트 가격이 대략 얼마인가요 💰가격
2. 뼈이식 임플란트 꼭 해야 되나요
3. 임플란트 게임 공략법 (노이즈)

④ 📋 추천 글감
1. "부평 임플란트 비용, 진짜 견적 공개" — 근거: 30일 발행 33건(🟢) + 💰가격 질문 매칭

⑤ ⚠️ 한계 & 다음
발행 100+는 "100건 이상(정확값 모름)". 다음: 이 키워드로 /블로그 진행할까요?
```

- **① 한 줄 결론**: `[쓸 수 있다 / 조건부 / 피하라]` 중 하나로 시작 + 근거 1문장. 총 80자 이내.
- **② 표**: 최대 8행. 🟢 황금(easy) / 🟡 진입가능(ok) / 🔴 포화(hard). 검색량 null → "측정미만", 발행 exact 아님 → "100+". keyword/topic만 조사했으면 ③은 헤더 + "(키워드 축만 조사)" 1줄, question만이면 ②를 같은 방식으로.
- **③ 질문**: 첫 줄에 갭 밴드 한글 + `(수요 {kin_total} ÷ 공급 {blog_1m_count})`. 질문 최대 10개, 상업마커는 끝에 💰. 동음이의·무관 질문은 "(노이즈)" 라벨을 붙이되 **목록에서 빼지 않는다**(수집 사실 보존).
- **④ 글감**: 1~3개. 각 항목마다 근거 숫자·판정을 **반드시** 인용.
- **⑤ 한계 & 다음**: 측정 한계 1~3줄 + 다음 행동 1줄(재조사 각도 또는 `/블로그` 진행).

**보고서 금지 3종**: ① 숫자 근거 없는 "좋아 보입니다/추천드립니다" 단독 판정 ② 도구가 안 준 검색량·CPC·발행량 수치 작성(null은 "측정미만") ③ 표 없이 키워드를 문단으로 풀어쓰기.

## 판정 읽는 법 (사용자가 물으면 이렇게 설명)

**축 ① 진입 (`entry`)** — 최근 30일에 그 키워드로 올라온 블로그 글 수. 적을수록 우리 글이 위로 간다.

| 값 | 뜻 |
|---|---|
| `easy` | 30일 발행 **10건 미만** — 아주 좋다 |
| `ok` | 30일 발행 **100건 미만** — 해볼 만하다 |
| `hard` | 30일 발행 **100건 이상** — 포화 |
| `null` | 조회 실패 (판정 불가라고 솔직히 말한다) |

**축 ② 단가 (`cpc_band`)** — 이번 조사에 나온 **후보들끼리 비교**한 상대 위치(상위 25% = `high`).

> ⚠️ 업종마다 CPC 절대값이 딴판이라(임플란트 3만원 vs 수제캔들 1천원) 고정 기준선을 쓰지 않는다.
> **절대값이 궁금하면 `naver_cpc_krw` 원본을 보라** — 항상 같이 나온다.

**축 ③ 미충족 갭 (`gap_band`, question 모드)** — "묻는 사람은 많은데 답한 글은 적은가"를 주제 단위로 판정.

| 값 | 뜻 |
|---|---|
| `wide` | ★황금 — 질문은 쌓였는데 답 글이 적다. 이 주제로 쓰면 먹힌다 |
| `moderate` | 해볼 만한 갭 |
| `narrow` | 갭 좁음 — 이미 답 글이 꽤 있다 |
| `saturated` | 포화 — 질문도 많고 답 글도 넘친다 |
| `low_demand` | 묻는 사람 자체가 적다 |
| `unknown` | 조회 실패 (판정 불가라고 솔직히 말한다) |

> ⚠️ **질문 개별 점수는 없다** — 지식iN API가 답변수를 안 줘서 갭은 **주제(seed) 단위**로만 판정한다. "이 질문이 1등"이라고 지어내지 말 것. 질문별로는 헤드키워드·상업마커(💰가격/비용/추천/근처 등)만 참고.

## 후속 액션 (결과를 주고 끝내지 말 것)

조사 후 **한 줄 제안**까지 한다.

| 상황 | 제안 |
|---|---|
| `easy`/`ok` + `high` 있음 | "**○○로 쓰세요.** 바로 글 쓸까요?" → 사용자가 원하면 `/블로그` 안내 |
| 전부 `hard` | "**더 좁혀야 합니다.** 지역명(검단·부평)이나 상황어(가격·후기·비용)를 붙여 다시 조사할까요?" |
| `hard` + `high` 만 있음 | "**단가는 최고인데 못 들어갑니다.** 그 키워드의 롱테일(예: `임플란트` → `검단 임플란트 비용`)로 파생 조사 권장." |
| `gap_band=wide` + 💰 질문 있음 | "**이 질문들로 글 쓰세요.** 실제 고객 질문을 소제목(H2)으로 쓰면 지어낸 질문보다 낫습니다." → `/블로그` 안내 |
| question 결과 노이즈 과다 | "**주제어가 동음이의입니다.** '○○ + 업종어'로 좁혀 재조사할까요?" |

## 함정

- **"검색량 많은 게 좋은 키워드"가 아니다.** 검색량 24만짜리 `치과`는 포화라 버리고, 5천짜리 `부평임플란트`를 고르는 게 이 도구의 존재 이유다. 사용자가 검색량만 보고 고르려 하면 말려라.
- **`_ref.honeypot_score`(꿀통 10~17)는 참조값이다.** 로워드와 대조하려고 남겨둔 것이고 **판정에는 안 쓴다.** 이걸 근거로 추천하지 말 것.
- **발행량은 근사다.** 네이버 *블로그 검색 인덱스* 기준이지 실제 발행 총량이 아니다(`basis.entry`에 출처가 찍혀 나온다). 경계값(9건 vs 11건) 근처에서는 뒤집힐 수 있다고 솔직히 말해도 된다.
- **조회 결과는 자동으로 회사 DB에 누적된다.** 네이버는 과거 CPC를 안 주므로, 쓸수록 "이 키워드가 비싸지는 중인가"를 우리만 알게 된다. 많이 쓰는 게 이득이다.
