# 세션 저널 적재 (팀원용)

이번 작업의 맥락을 **노션 팀 세션 저널 DB**에 1행 기록한다.
목적: 무엇을·왜 했는지 구조화해 누적 → 나중에 검색·정리·인수인계가 쉽게.

> 팀원용: 복잡한 정본 라우팅은 빼고 **"오늘 작업 5섹션 기록"** 에 집중한다.

## 적재 여부 판단 (먼저 자문)

이번 작업에 **핵심 결정 / 만든 것(산출물) / 배운 점** 중 하나라도 있었는가?
- YES → 적재.
- NO (단순 질문·조회만) → "이번 작업은 기록 대상이 아닙니다" 1줄 보고 후 종료.

## 적재 방법

`notion-create-pages` 호출:
- **parent**: `{"type":"data_source_id","data_source_id":"67bf5daa-64e2-4745-85b6-c67e4a231b44"}`
  - (줄갭 "팀 세션 저널" DB — 마스터 허브 하위. 직원이 이 DB에 노션 권한이 있어야 적재됨)
- **properties**:
  - `세션` (title): 이번 작업 한 줄 제목
  - `date:날짜:start`: 오늘 날짜 (YYYY-MM-DD), `date:날짜:is_datetime`: 0
  - `유형`: 콘텐츠제작 / 조사 / 문서작성 / 기타 중 1개
  - `한줄 요약`: 이 작업에서 가장 중요한 것 1줄
  - `산출물`: 만든 것(링크·파일명)
- **content (body) = 표준 5섹션 고정**:
  ```
  ## 배경 / 요청
  무엇을 하려 했는가 (1~2줄)

  ## 핵심 결정 + 근거
  무엇을 정했고 왜

  ## 학습 · 함정
  하면서 알게 된 것, 다음에 조심할 것

  ## 산출물
  만든 콘텐츠 / 문서 / 링크

  ## 미해결 · 다음
  다음에 이어서 할 것
  ```

## 캡처 원칙 (노이즈 차단)

- **담아라**: 핵심 결정·근거 / 만든 것 / 배운 점.
- **빼라**: 도구 실행 로그 / 중간 과정 / 뻔한 것.
- **형식 일관성 = 매번 같은 5섹션** (나중에 검색·정리가 쉽다).

## 완료 후

적재된 노션 페이지 URL을 보고한다. (텔레그램 알림은 보내지 않는다)
