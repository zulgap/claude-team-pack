# 썸네일 — 대본 → 기획 3안 → 대화 수정 → 생성 (팀원용)

대본을 주면 **미스터비스트 후킹 공식**으로 썸네일 기획안 3개를 뽑고, 네가 대화로 고른/고친 뒤 **완성 썸네일**(배경 + 정확한 한글 텍스트)을 만들어 준다.

> **핵심**: 텍스트는 GPT가 그림 안에 그리면 한글이 깨진다 → 이 스킬은 **배경만 GPT Image 2로 생성 + 글자는 HTML 오버레이**로 얹어 **픽셀 정확**하게 만든다.
> **제디(jedi) 도구가 활성일 때만 동작** — 개인 `JEDI_TOKEN` 필요.

## 입력 판별 (먼저 이것부터)

| 입력 | 처리 |
|---|---|
| **대본** (필수) | STEP 1~3으로 기획 3안 생성 |
| **채널명** (선택) | `presets/<채널>.md` 읽어 코어 위에 적용 (동봉 예시 프리셋 참조) |
| **인물 사진** (선택, 로컬 경로/첨부) | 배경 생성 시 그 인물 합성 (얼굴 유지) |

채널 프리셋 없으면 범용 기본값으로 진행. 프리셋 있으면 **반드시 먼저 읽고** 타깃/갭언어우선순위/배경무드/금칙/인물규칙을 적용.

## 🧭 알고리즘 (대본 → 기획안 3안)

### GATE 0 — 5대 기반 법칙 (모든 안이 먼저 통과, 위반 시 폐기)
1. **극단성**: 평범→흥미→극단 사다리 (앞마당50시간❌→케첩속50시간✅ / Spent→Survived)
2. **기대치 계약**: 대본이 실제로 지킬 약속인가? **클릭만 끌고 못 지키는 낚시 금지**
3. **단순성**: 3초·한 메시지·요소 욕심 없음
4. **Wow**: 남들 못 하는 각 1개
5. **핵심요소**: 썸네일 = 영상 설계도 (산출물 아님)

### STEP 1 — 대본 분석
- 타깃(프리셋 있으면 그 페르소나) · 핵심 메시지 1문장 · **의외 소재 후보 3개**(경험자/재도전/실패후/통념깨기 우선) · 감정축(불안·기대·놀라움)

### STEP 2 — 기획안 3개 생성 (대본 성격에 따라 갭언어 전략 적응 선택)
각 안 = 아래 4요소. **대본 성격에 맞춰 서로 다른 갭언어 유형 3개**를 AI가 선택(고정 분산 아님):

| 요소 | 규칙 |
|---|---|
| **제목**(영상용) | 경험자 명시 + 페르소나(예 "33세 약사") + "실제 사례". 결론 스포·과도 디테일·AI체 금지 |
| **메인카피**(썸네일 후킹) | **9~13자 구(phrase)**, 질문형/미완성형, **스포 금지**, 숫자·'이것' OK(정답 스포만 금지). 강조할 1단어(또는 아랫줄 전체) 표시 |
| **서브카피**(하단 자막) | **14~22자**, 사례로 고정 — 페르소나/시간("N년 만에")/상황 중 2~3개 |
| **배경컨셉** | 무드로 신뢰(상담실/대화/노트 등). 프리셋 인물 규칙 적용 |
| **레이아웃**(A/B/C) | 대본 성격으로 자동 선택 (아래 표) |

**갭언어 8유형**(대본 성격에 맞춰 3개 선택): 통념깨기 / 손실회피 / 사회적증거 / 구체성N가지 / 신뢰성 / 공감형 / 끝장 / 이유·방법. 각 안에 유형 태그를 붙여 제시.

### 레이아웃 3종 (실측 12장 기준 — 표본 상세는 PROVENANCE.md)

| 코드 | 레이아웃 | 언제 이걸로 (대본 성격) |
|---|---|---|
| **A** | 하단 2줄 임팩트 (기본) | **대부분의 롱폼** — 사연·상담·정보. 여러 등장인물/상황극이면 말풍선 부품 켬 |
| **B** | 좌텍스트·우인물 | 1인 인터뷰·경험담·talking-head |
| **C** | 상단센터 (쇼츠 9:16) | 세로 영상·쇼츠 |

- 실측 스펙: 글자 = 프레임 높이의 16~18%(대형 2줄), **1줄 흰색 / 2줄(또는 핵심어) 강조색**, 검은 외곽선.
- **강조색**은 채널 프리셋이 정함. 프리셋 없으면 노랑 기본.

### STEP 3 — 10초 검수 체크리스트 (3안 각각 자동 통과)
- [ ] 썸네일만 봐도 경험자/상황이 한 번에 들어오나?
- [ ] 정답/결론을 말하고 있지 않나?
- [ ] 제목은 썸네일보다 **딱 한 단계**만 더 설명하나?
- [ ] 메인/서브가 길이 규격(9~13 / 14~22자)을 지키나?
- [ ] 개인·민감정보 노출 위험은 없나?

**→ 기획 3안을 사용자에게 제시** (안별로 제목/메인카피/서브카피/배경컨셉/갭언어유형). "몇 번 안으로 갈까요? 고칠 부분 있으면 말씀해 주세요."

## STEP 4 — 대화 수정 → 확정
- 사용자가 고른 안 + 수정 요청("2번 메인 더 세게", "서브 짧게") 반영.
- 확정되면 아래 **생성** 진행.

## STEP 5 — 생성 (배경 + 인물 합성 + 텍스트 오버레이)

### 5-1. 배경 생성 (텍스트 없이 배경만)
`ext_generate_image` 호출 — **배경컨셉을 프롬프트로** (프리셋에 인물 규칙이 있으면 그대로 프롬프트에 주입). **"이미지 안에 글자 절대 넣지 말 것"** 명시. `n: 1`.
- **비율**: 레이아웃 A/B = `aspect_ratio: "16:9"` / 레이아웃 C(쇼츠) = `"9:16"`.
- 레이아웃 B는 인물이 **오른쪽**에 오도록(왼쪽은 어둡게·여백 = 텍스트 공간), C는 인물이 **중앙~하단**에 오도록 프롬프트로 유도.
- 반환 `image_url`을 배경으로 사용.

### 5-2. 인물 합성 (사진 준 경우만)
- 로컬 사진 → 먼저 업로드: `node <이 스킬 폴더>/../이미지/scripts/upload-image.mjs "C:/경로/사진.jpg"` (마지막 줄 = URL).
- `ext_generate_image` `mode: "compose"` 또는 `"style_ref"` + `input_images: [{url, role}]` + `preserve_fidelity: true` 로 배경에 인물 합성.

### 5-3. 텍스트 오버레이 렌더 (픽셀 정확 한글)
`compose-thumbnail.mjs`가 base CSS + 선택 레이아웃을 조립해 백엔드로 렌더:
```
node <이 스킬 폴더>/scripts/compose-thumbnail.mjs \
  --layout A \                    # A(하단2줄·기본) / B(좌텍스트·우인물) / C(쇼츠9:16)
  --bg "<배경 image_url>" \
  --title "월 500 노처녀들이|**상향혼**에 빠지는 과정" \   # | = 줄바꿈, **강조** = 색
  --subtitle "확정 서브카피" \                            # 선택
  --acc yellow \                  # 강조색: yellow / red / pink — 채널 프리셋이 지정
  --role <역할라벨> \              # 우상단 (역할) 라벨 — 진행자 채널 (선택)
  --channel <채널명> \            # 우하단 워터마크 (선택, 프리셋의 워터마크 값)
  --host "<진행자 얼굴 image_url>" \   # A 레이아웃 우측 인물 (선택)
  --quote "제발 저리가!!@22,14"    # 인물 위 말풍선, x,y=%, ! 접두=빨강 (상황극, 선택·반복가능)
```
- **레이아웃 자동선택**: STEP 2에서 정한 A/B/C를 `--layout`으로 넘김.
- **강조색·역할라벨·외곽선**은 채널 프리셋의 compose 인자를 그대로 따름.
- 스크립트 마지막 줄 출력 = **완성 썸네일 image_url**.
- 사용자에게 전달: `✅ 썸네일 완성 (레이아웃 {A/B/C})\n{image_url}`

## 프리셋 (채널별 세부 규칙)
- `presets/` 폴더에 채널당 1파일. frontmatter `preset_slots`의 빈칸을 채우는 치수 카드다 (동봉 예시 프리셋 1개 참조).
- 다른 채널은 프리셋 파일 1개 추가하면 코어 무수정으로 적용됨. 새 회사 설치 시 작성 규약 = `docs/skill-packaging-spec.md` §3 설치 인터뷰.

## 기준 정본 (이 스킬의 근거)
- 출처·실측 표본 상세는 `PROVENANCE.md` 참조 (심사 grep 제외 영역).

## 안 되는 경우
- `ext_generate_image` 없음 → `JEDI_TOKEN` 미등록. install 재실행/사장님께 토큰 요청.
- compose 스크립트가 `render-thumbnail 엔드포인트 없음(HTTP 404)` → 백엔드 미배포. 사장님/개발자에게 문의.
- 한글이 깨진 배경이 나옴 → STEP 5-1에서 "글자 넣지 말 것"을 안 넣었을 가능성. 텍스트는 5-3 오버레이가 담당하므로 배경엔 절대 글자 X.
